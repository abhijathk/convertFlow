## Vision

### 10x Check
The 10x version of Utilities isn't a tab you navigate to — the utilities become ambient context
everywhere. Inline token counts in the Editor gutter, a cost estimate badge in Convert's status
bar, auto-validation on every file open. Users never have to leave their current context to get
a number. The Utilities tab becomes the shared calculation engine that powers the whole app,
surfacing answers where the user already is.

**Note: the 10x ambient features (Editor gutter, Convert status bar, auto-validation) are NOT
in scope for this plan. They are downstream of the Registry API and will be planned separately
once the registry is stable.**

### Landscape Position
7+ standalone token calculator sites exist (tokencalculator.com, pricepertoken.com, tokencost.app
etc.). They're excellent at their single job. ConvertFlow's advantage is not depth — it's workflow
continuity. The user's dataset is already open. Auto-prefill eliminates the copy-paste round-trip
that makes external tools feel like a detour.

## Architecture Concerns (from review)

1. **Lazy-loading**: Utility modules must be lazy-loaded, not bundled upfront. The seedrandom,
   base64, and URL encoding libraries add avoidable weight for users who never open Utilities.
   Use `import.meta.glob` (the correct Vite API for dynamic tool loading):
   ```typescript
   const toolLoaders = import.meta.glob('./tools/*.ts');
   // Usage: const mod = await toolLoaders[`./tools/${id}.ts`]();
   ```
   This gives Vite static analysis of the glob while keeping each tool as a separate lazy chunk.
   Do NOT use template-literal `import()` with a variable path — Vite cannot split those.

2. **Keyboard shortcut**: Wire ⌘4 in `src/components/TabStrip.svelte` keybinding map
   alongside the existing ⌘1/⌘2/⌘3 entries. Also add to the command palette.

3. **Auto-prefill size guard**: JSONL files can be 50MB+. Auto-prefill truncates at 100KB
   with an inline warning: "File too large to auto-load — showing first 100KB."
   A "[Load full file]" button (user-initiated, not auto) loads the full `activeFile.content`.
   This is justified as an essential safety valve — without it, users with large files would
   silently get truncated results with no way to fix it.

## Implementation Notes

### Auto-prefill (Cherry-pick 1)

**State model** (per-tool, not global — each tool tracks its own prefill state independently):
```typescript
interface ToolInputState {
  primaryInput: string;
  autoPrefilled: boolean;       // true = input came from Editor file this activation
  prefillSourceFileId: string | null;  // which file, for JSONL validator navigation
  prefillTruncated: boolean;    // true = content was capped at 100KB
}
```

`autoPrefilled` is per-tool so switching from Token Estimator (already prefilled) to Dataset
Splitter (not yet opened) triggers a fresh prefill for the new tool.

**Activation definition**: "activation" = each time the Utilities tab gains focus (tab click or
⌘4). On activation, for each tool where `autoPrefilled = false` AND the primary input is empty,
prefill is attempted. A tool that was already manually edited (`autoPrefilled` cleared) is left alone.

**What "primary input" means per tool** (informal term, defined here):
- Multi-input tools: the main content textarea (Dataset Splitter's JSONL area, Regex Tester's
  test-string field, NOT the pattern field). Single-input tools: the only textarea.

In `utilitiesState.ts`, on activation, for each eligible tool:
1. Read `activeFile` from `editorState`. If null OR `activeFile` was deleted (check against
   `editorState.openFiles`), skip silently.
2. If `activeFile.content.length <= 100_000`: set input, set `autoPrefilled = true`,
   set `prefillSourceFileId = activeFile.id`, set `prefillTruncated = false`
3. If `activeFile.content.length > 100_000`: set first 100KB, set `autoPrefilled = true`,
   set `prefillSourceFileId = activeFile.id`, set `prefillTruncated = true`,
   show inline warning + "[Load full file]" button (loads full `activeFile.content` on click)

**Manual edit rule**: On any change event in the primary text input, clear `autoPrefilled`,
clear `prefillSourceFileId`. The field is now user-owned for this session.

**"Go to line" button visibility rule (single authoritative definition)**:
The button renders IFF `prefillSourceFileId !== null` at the moment the results panel renders.
Since `prefillSourceFileId` is cleared on any user edit, the button disappears as soon as the
user modifies the input — no separate `autoPrefilled` check needed at render time.

### Registry API (Cherry-pick 3)

`src/lib/utilities/registry.ts` exports:

```typescript
export interface UtilityPayload {
  input: string;               // primary text content
  options?: Record<string, unknown>;  // tool-specific settings
}

export interface UtilityResult {
  ok: boolean;
  data?: unknown;              // tool-specific output
  error?: string;              // present when ok = false
  metadata?: Record<string, unknown>;  // counts, stats, etc.
}

export async function runUtility(id: string, payload: UtilityPayload): Promise<UtilityResult>
```

`runUtility` does NOT throw — it catches all errors and returns `{ ok: false, error: string }`.
Callers in Editor/Convert should check `result.ok` before using `result.data`.

Import pattern: `import { runUtility } from '../lib/utilities/registry'` — no Svelte store
dependency, works from any context.

### JSONL validator navigation (Cherry-pick 4)

JSON/JSONL Validator's `UtilityResult.data` shape:
```typescript
interface ValidationResult {
  valid: boolean;
  errors: Array<{
    line: number;
    message: string;
    editorFileId: string | null;   // set when autoPrefilled = true at run time
  }>;
}
```

The validator captures `prefillSourceFileId` at run time (snapshot — not reactive) and
attaches it to each error object.

Result panel renders a "Go to line N in Editor" button for each error when `editorFileId`
is non-null. The button:
1. Calls `selectFile(editorFileId)` from editorState (switches to Editor tab + selects file)
2. Calls `setCursorPosition(editorFileId, line, 0)` — use this canonical signature throughout

**Pre-implementation check**: confirm `setCursorPosition(id: string, line: number, col: number)`
exists in `editorState.ts`. If not, add it. Do not use an object-form `setCursor({ line, col })`
call site — pick one signature and use it everywhere.

The "Go to line" button is hidden (not rendered) when `autoPrefilled = false` at the time
results are rendered (regardless of whether auto-prefill fired earlier in the session).

### Dataset Splitter (Cherry-pick 2)

Inputs: text area (JSONL), train%/val%/test% numeric inputs (must sum to 100 — validated live
with inline error), optional seed text field.

**Shuffle**: Use Fisher-Yates (unbiased):
```typescript
function shuffle<T>(arr: T[], rng: () => number): T[] {
  const a = [...arr];
  for (let i = a.length - 1; i > 0; i--) {
    const j = Math.floor(rng() * (i + 1));
    [a[i], a[j]] = [a[j], a[i]];
  }
  return a;
}
```
When seed field is non-empty: `import seedrandom from 'seedrandom'`, pass `seedrandom(seed)` as `rng`.
When seed is blank: pass `Math.random` as `rng`.

**Dependency**: `npm install seedrandom @types/seedrandom`. Confirm in package.json before
starting. API: `const rng = seedrandom('my-seed'); rng()` returns a float [0,1).

**Split formula** (avoids off-by-one from float rounding):
```typescript
const n = lines.length;
const trainEnd = Math.floor(n * (trainPct / 100));
const valEnd = trainEnd + Math.floor(n * (valPct / 100));
// test gets the remainder — guarantees all lines are included
const trainLines = lines.slice(0, trainEnd);
const valLines = lines.slice(trainEnd, valEnd);
const testLines = lines.slice(valEnd);  // remainder, not a computed slice
```

**Edge cases**:
- `lines.length === 0`: return `{ train: '', val: '', test: '' }` with a warning "No lines to split."
- `valPct === 0` or `testPct === 0`: the corresponding output string is empty. Still create the
  "Send to Editor" file with empty content (so the user can see the file exists and add to it).
  "Copy val" / "Copy test" buttons are greyed out when the split is empty.

Output: `{ train: string, val: string, test: string }` where each is `lines.join('\n')`.

**"Send to Editor" naming**:
- When `prefillSourceFileId !== null`: derive base name from the prefill source file's name
  without extension (e.g., `'dataset.jsonl'` → base = `'dataset'`)
- When not prefilled: base = `'dataset'`
- File names: `{base}_train.jsonl`, `{base}_val.jsonl`, `{base}_test.jsonl`
- Call `openFile(name, content)` from `editorState.ts` — function signature: `openFile(name: string, content: string): string` (returns new file id)
- After opening all three, set `activeFileId` to the train file id (first created)

### Lazy-loading utility modules

```typescript
// src/lib/utilities/registry.ts

// Each tool module must export a default UtilityToolModule:
export interface UtilityToolModule {
  id: string;
  name: string;
  category: 'llm' | 'data' | 'text' | 'validation' | 'encoding' | 'utility';
  run: (payload: UtilityPayload) => Promise<UtilityResult> | UtilityResult;
}

// Vite builds one lazy chunk per file in ./tools/ at build time
const toolLoaders = import.meta.glob('./tools/*.ts') as Record<
  string,
  () => Promise<{ default: UtilityToolModule }>
>;

const toolModuleCache = new Map<string, UtilityToolModule>();

// loadTool: internal only — runUtility wraps it and catches all errors
async function loadTool(id: string): Promise<UtilityToolModule> {
  if (!toolModuleCache.has(id)) {
    const loader = toolLoaders[`./tools/${id}.ts`];
    if (!loader) throw new Error(`Unknown utility tool: ${id}`);
    const mod = await loader();
    toolModuleCache.set(id, mod.default);
  }
  return toolModuleCache.get(id)!;
}

// runUtility: public API — NEVER throws; always returns a UtilityResult
export async function runUtility(id: string, payload: UtilityPayload): Promise<UtilityResult> {
  try {
    const tool = await loadTool(id);
    return await tool.run(payload);
  } catch (e) {
    return { ok: false, error: e instanceof Error ? e.message : String(e) };
  }
}
```

Vite sees the static glob at build time and creates separate lazy chunks for each file in
`./tools/`. Each tool only loads when first activated. `runUtility` never throws — all errors
from `loadTool` and `tool.run` are caught and returned as `{ ok: false, error }`.

**Non-output tools**: All tools in the MVP table that have a result produce either a "Copy"
button or a "Send to Editor" button in the result panel. Copy calls `navigator.clipboard.writeText`.
Send to Editor calls `openFile(name, content)` and switches the app to the Editor tab.
Tools without a textual output (e.g., Token Estimator which produces counts + stats) show
the stats inline and offer a "Copy summary" action that copies a formatted text representation.
